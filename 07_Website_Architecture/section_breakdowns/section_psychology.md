## 01. Section 1: The Hero

### Psychological Objective
**Stop the scroll. Create a question.**

The visitor should feel: *"I don't know exactly what this is, but I want to."*

### Design Psychology
- **Extreme scale contrast** (96px headline / 18px sub) creates instant hierarchy
- **Dark, atmospheric background** communicates seriousness and depth
- **Minimal copy** communicates confidence — we don't need to explain ourselves
- **3D visualization** creates wonder — there is more here than meets the eye

### Copy Psychology
The headline should create a *cognitive gap* — the visitor knows their current state (spreadsheets are static) and can feel the implied new state (they could be alive) but doesn't yet understand how.

This gap creates the pull to keep scrolling.

### Design Specification
```
Background: Deep dark + gradient + particle field
Layout: Centered, single column, max-width 800px
Space above fold: Logo + Nav (56px) → Hero content
Hero content top margin: 160px from nav
Headline: 96px, weight 700, centered, gradient (white → 50% white)
Sub: 20px, weight 400, centered, opacity 0.6, max-width 560px
CTA pair: 48px height buttons, 16px gap, centered
Trust bar: 80px below CTAs, 4 company logos, desaturated
```

## 02. Section 2: The Problem

### Psychological Objective
**Mirror the visitor's reality. Make them feel seen.**

The visitor should feel: *"This is exactly what I deal with."*

### Why This Section Matters
Most SaaS brands rush to their solution.
The brands that convert at the highest rate spend time validating the problem first.
When a visitor feels understood, their resistance to the solution drops.

### Copy Pattern
```
You've built elaborate systems.
You've hired analysts.
You've tried automation tools.

And every Monday morning, someone is still staring at a spreadsheet,
manually doing what should have been done automatically.

The problem was never your people.
It was your tools — they were built before intelligence existed.
```

### Design Psychology
- **Typography-led section** — no images, all words. Maximum emotional impact.
- **Line breaks are deliberate** — each line is a breath, a beat
- **White space amplifies weight** — few words in large space feel more significant
- **Color shift** — this section has a slightly different background tint to signal "we're going deeper"

## 03. Section 3: The Product Reveal

### Psychological Objective
**Create the "aha" moment. The first glimpse of relief.**

The visitor should feel: *"Oh. This is different."*

### Reveal Mechanics
Not a product screenshot. A product *demonstration*.
The UI doesn't sit static — it runs. Data changes. Intelligence responds.

### Three-Stage Reveal Concept
```
Stage 1: "The Before"
An ordinary spreadsheet. Familiar. Recognized.
Copy: "This is what your data looks like today."

Stage 2: "The Activation"
SheetFlow AI logo appears. A connection forms.
Copy: "This is the moment it changes."

Stage 3: "The After"
The spreadsheet is now live — changes detected, intelligence visible.
Automations fire. Reports generate. Alerts dispatch.
Copy: "This is your data on SheetFlow AI."
```

### Scroll-Linked Animation
The transition from Before to After is tied to scroll position.
The visitor controls the speed of the transformation.
This is more memorable than watching it automatically — they did it.

## 04. Section 4: Feature Architecture

### Psychological Objective
**Build trust through specificity. Show depth without overwhelming.**

The visitor should feel: *"There's real substance here."*

### The Bento Strategy
Rather than a flat feature list, features are presented as a system.
Some features are large (core capabilities) — others are small (supporting capabilities).
The visual hierarchy communicates relative importance.

### Feature Bento Layout

```
┌─────────────────────────────┬──────────────────┐
│                             │                  │
│   AUTONOMOUS MONITORING     │   SMART ALERTS   │
│   [animated dashboard]      │   [alert card]   │
│   Your data watches itself. │                  │
│                             │                  │
├──────────────┬──────────────┴──────────────────┤
│              │                                  │
│  LIVE        │   NATURAL LANGUAGE COMMAND       │
│  PIPELINES   │   [command palette demo]         │
│              │   Ask SheetFlow AI anything.     │
└──────────────┴──────────────────────────────────┘
```

### Copy Principle for Features
Each feature card leads with the **consequence**, not the capability.

*Wrong*: "Monitors your spreadsheets for changes"
*Right*: "Your data catches its own errors before you do"

## 05. Section 5: Social Proof

### Psychological Objective
**Transfer trust from existing users to the visitor.**

The visitor should feel: *"Teams I respect use this. I can trust it."*

### Social Proof Hierarchy (Most to Least Persuasive)
1. Specific statistics with attribution ("27 hours saved per week — Ops team at [Company]")
2. Named testimonials with photo and role
3. Company logos (recognition transfer)
4. Anonymous statistics ("thousands of teams...")

### The Quote Architecture
```
The best testimonial for SheetFlow AI has four elements:
1. The state BEFORE  ("We were spending 3 days each month on reporting...")
2. The specific change ("Now SheetFlow AI generates our board report every Friday morning...")
3. The unexpected benefit ("...but the bigger win was catching a $40K billing error.")
4. Attribution (Full name, title, company — no hiding)
```

## 06. Section 6: Pricing

### Psychological Objective
**Remove the financial anxiety. Make the "yes" feel obvious.**

The visitor should feel: *"This is clearly the right tier for me."*

### Pricing Psychology Principles

**Anchoring**: Show the highest tier first (or in the center) so the lower tiers feel accessible.

**Loss framing**: Frame free plan limitations as what you're missing, not what you have.

**Decoy pricing**: The middle tier should feel like the obvious choice.

**Value framing**: Always present price alongside value, not alone.
- Not: "$79/month"
- But: "$79/month — less than one analyst hour"

## 07. Section 7: Final CTA

### Psychological Objective
**Crystallize the decision. Remove every barrier.**

The visitor should feel: *"I've been convinced. Now I just need to act."*

### The Final CTA Design Principle
The final CTA should feel like a **completion**, not a repetition.

The hero said: "Your data, finally alive."
The final CTA echoes this, but from a position of completion:
"Now you know what's possible. The next step is yours."

### Final CTA Architecture
```
[Subtle background — mirrors hero gradient]

Kicker: "Start today. See results this week."

Headline: "Your operation is waiting."
          (short, declarative, no explanation)

Single CTA: "Start Free — No Credit Card"

Below CTA: "Setup in 5 minutes. Cancel anytime."
            (removes last friction points)

Social signal: "Join 2,400+ teams already running smarter."
```
